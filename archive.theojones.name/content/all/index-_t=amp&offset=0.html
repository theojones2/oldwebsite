Why I ad block

2 min read

There have been a few recent controversies about internet adblocking. The most prominent of these has been Apple's decision to allow ad-blocking software on the App Store for Ipads. I will admit to running adblocking software on my computer and on the pfsense box I use as a router. I realsize that there are some issues with this. I get that there is an impact on the revinue of the websites I visit, as many websites use cost per impression ads that pay out based on the number of visitors to the website. I know that there have been cases where ad blockers have abused their control over the blocklists inorder to make profit. Adblock Plus, for instance, will de-list the ads on a website -- for a fee. However, there are some reasons why I use adblocking software on my computer

  1. Tracking and privacy. Advertisers use their networks to monitor what websites the viewers of the ads go to, and keep a databse of this tracking. They use this information about web browsing habits to create a profile of each user, and to guess at a wide range of other personal information. This information is used to "target" the ad to the users, and is quite often sold to other companies. This is a substintial violation of privacy and one of the main reasons I ad block. 
  2. Computer performance issues. A lot of ads contain flash/javascript code that is quite ram and CPU hogging. 
  3. Secutity issues. There have been cases where ads have contained malware. Which isn't surprising, considering that the core of internet targeted advertising is to load untrusted content onto the website you are viewing. Due to the need to keep advertising "targeted" this is pretty much unavoidable under the current internet marketing model. 
  4. These problems are at the core of the internet advertising model. Unlike a dead-tree publication, you can't effectively target on geographic location, or the like. So, the much more intrusive methods of advertising become unavoidable.

Should I Upgrade to Mac OS X El Capitan?

"One thing in El Capitan you might want to pay attention to is a new security feature called System Integrity Protection (SIP), which has the potential to break some apps. In short, Apple introduced it as a way to limit even the level of access that root users and processes have, and to protect core components of the OS. That means there are some things you won’t be able to do, even if you have and use root. You can read the nitty gritty over at Ars Technica. This also means some applications that leverage root won’t work anymore—one notable example is Bartender, before it was recently updated with El Capitan support. It’s not a huge problem, and you can disable it, but in all reality most people won’t need to and shouldn’t have to. However, some programs may need to be updated before they’ll work—and if you’re using some long-abandoned apps that have worked up to this point and they break, well, they could be dead for good, and you’ll have to make the call to disable it or find an alternative."
